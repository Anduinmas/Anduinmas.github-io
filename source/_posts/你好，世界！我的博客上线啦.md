> 📝 写在前面：花了几个晚上，终于把属于自己的一隅小天地搭建好了。想在这里记录一些东西——技术的、生活的、读书的、瞎想的。写下来，才算真正拥有过。

## 为什么要写博客？

这个问题我问过自己很多次。

市面上有掘金、知乎、微信公众号，似乎不缺一个发声的地方。但每次想在那些平台写点什么，总有一股无形的不自在——**排版要迎合流量、标题要追求点击、内容要追逐热点**。

我只想安安静静地写点东西。

所以有了「**一隅小记**」。一个完全属于我的角落。不用考虑 SEO 标题怎么写，不用纠结封面图吸不吸引人。写技术就写技术，写生活就写生活，写得长就长，短就短。

## 怎么搭起来的？

对于一个只会简单前端的人来说，搭博客其实比想象中简单。

{% note info %}
**技术栈**：Hexo + Butterfly 主题 + GitHub Pages，**完全免费**。
{% endnote %}

流程大概是这样的：

1. 装好 Node.js 和 Git（这两个是基础）
2. 用 `hexo init` 初始化项目
3. 选一个好看的主题——Butterfly，一眼就爱上了它的文艺气质
4. 配置一堆功能：搜索、暗黑模式、RSS、评论、访问统计
5. 部署到 GitHub Pages，绑定域名
6. 开始写第一篇文章

全程花了大概一个下午。最花时间的不是技术，而是**想好自己到底要写什么**。

## 打算写什么？

我给自己划了四个方向：

| 方向 | 内容 | 频率 |
|------|------|------|
| 💻 **技术踩坑** | 工作中学到的、踩过的坑 | 有空就写 |
| 📚 **读书笔记** | 读完一本书，写一篇笔记 | 随缘 |
| ☕ **生活随笔** | 日常的温馨瞬间 | 有感就记 |
| 📝 **职场复盘** | 定期停下来看看 | 季度更新 |

不设硬性目标——**灵感来了就写，忙了就停下**。博客应该是生活的一部分，不是生活的负担。
